項目預算 vs 員工費用
該怎麼分？

Ray 跟 Shawn 在做批發 / 快閃項目的預算編列時遇到分類問題。本文檔提供 3 條方案 + 推薦方向，看完一起拍板再進工程。

遇到的兩個問題

問題 1
編項目預算時，**該不該把這段期間的「人員成本」（項目相關差旅 / 招待 / 樣品試做工時）算進項目預算**？
問題 2
員工申請費用時，**項目相關差旅怎麼跟一般個人報銷分開？** 以及，怎麼確認該筆費用**沒超出當初編預算時的標準**？

表面看是兩個問題，本質是同一個：項目實際 P&L 該包含哪些成本、員工申請費用怎麼歸屬。

先釐清現況分類（Ray 提醒的重點）

我們之前定義費用申請是兩類，但**「業務費用申請」當初的定義是「幫供應商請款」**（B2B 代墊 / 供應商請款），不是員工因項目產生的差旅。所以現在這個項目差旅根本沒有正確的 bucket：

類別誰申請對象是誰典型例子
① 個人報銷員工員工自己個人出差餐費 / 通勤 / 自費耗材
② 供應商請款（既有「業務支出」）採購 / 業務外部供應商 / 代墊對象工廠尾款 / 樣品費 / 外包設計費 / 場地費
③ 項目員工差旅 ★ 新類別員工員工自己（但因項目而花）場勘出差 / 開幕值班 / 客戶拜訪

過去我們把③ 硬塞進①「個人報銷」走，導致 — 項目成本看不到、員工自己分不清是工作還是私事。

本文件聚焦 ③ 怎麼處理
① 個人報銷（員工日常） + ② 供應商請款（B2B）流程都 OK 不動，純粹討論：因項目而生的員工差旅，怎麼納管 + 怎麼跟項目預算對齊。

三條路線

A 兩本帳分開（現況） 最沒紀律
項目預算只算硬成本（採購 + IP 版權 + 物流 + 場地）。所有人員 / 差旅 / 招待 → 全部歸 OPEX。員工申請費用一律走「個人報銷」流程，跟項目脫鉤。
✓ 員工最簡單 — 不用想項目代碼
✗ 項目 ROI 算不準 — 一個快閃店花了多少人力差旅，事後查不到
✗ OPEX 黑盒 — 月底 OPEX 高了，不知道是哪個項目造成的
✗ VAM 800 萬看不清 — 看到的「項目淨利」是被高估的（人力成本沒扣）
B 雙軌制 — 一筆費用兩個歸屬 推薦
員工申請費用時，多選一格「關聯項目」（可選）。有填 → 同時計入「項目實際成本」+「OPEX 差旅池」；不填 → 只算 OPEX。Shawn 編項目預算時，分項加一行「人員 / 差旅」做預估。
✓ 項目 P&L 真實 — Ray 看到的是含人力成本的淨利
✓ OPEX 流向可查 — 月底能看「OPEX 差旅 → 哪些項目吃掉了」
✓ 員工負擔小 — 只多選一格，不用走兩套流程
✓ 會計界 standard — Xero / QuickBooks / SAP 都這樣設計（cost center + nature 雙軸歸屬）
✗ 員工要記項目代碼 — 但系統 dropdown 帶出來，非完全憑記憶
C 純項目歸屬制 最嚴
所有費用必須掛項目代碼，沒項目 → 系統拒收。員工日常拜訪 / 內部會議 / 學習出差也得想辦法掛上去。
✓ 紀律最強 — 沒項目不能花錢
✗ 太僵化 — 員工 onboarding / 跨項目培訓 / 一般辦公差旅 真的沒項目可掛
✗ 員工會「亂掛項目」湊數 — 反而把資料弄髒，跟現況沒兩樣
✗ 對小公司過重 — 蕪解 15 人不需要這麼緊

為什麼推 B

三條理由：

  • 會計界共識：成本歸屬走兩軸是常識 — 「性質」（差旅 / 餐費 / 招待）+「歸屬」（哪個項目 / 哪個部門 / 公司 OPEX）。一筆費用同時有兩個 label，不衝突。
  • 對員工負擔最小：費用申請已經有 5-6 個欄位，多一個 dropdown 不會更累。而且關聯項目可選不必填，按下「無關項目」走 OPEX 也行。
  • VAM 800 萬看得清：Ray 在戰情室要看「批發貢獻多少淨利、快閃貢獻多少淨利」— 沒有人力成本的版本是假帳，要看真的。

B 方案具體怎麼跑

1編預算Shawn

進「項目預算編列」時，預算分項加一行 人員 / 差旅池：

預算分項預估金額備註

IP 版權費¥ 35,000本批 MG 折抵
物流¥ 18,000工廠直發 + 報關
場地¥ 40,000快閃才有
人員 / 差旅池 ★ 新增¥ 25,000場勘 2 趟 + 開幕 3 人
項目總預算¥ 358,000
2員工申請費用員工 / 詩彤 / 高維 etc.

費用申請 modal 加一格「關聯項目」（可選 dropdown）：

— 不選 = 走純 OPEX，不關聯任何項目
3楊子審批時楊子

楊子 inbox 看到費用時，同時顯示對應項目預算池剩餘額度：

費用類別： 差旅
關聯項目： BKK-JJK-快閃 2026
┗ 人員/差旅池 ¥25,000 / 已用 ¥18,200（73%）
┗ 此筆扣減後剩 ¥3,400 (14%)

[✓ 核准]　[退回]　[預算不夠 → 升級 Ray]
超預算邏輯
若申請金額 > 該項目池剩餘 → 楊子無法直接核准，必須走升級 Ray 流程（避免靜默超支）。
4月底結算 · 項目結案躍明 / Shawn

月底兩個視角同時可看：

  • 躍明月結報表：OPEX 差旅總 ¥XXX，其中 ¥XXX 來自項目（含關聯項目 breakdown）
  • Shawn 戰情室：項目實際 P&L = 採購 + IP + 物流 + 場地 + 人員（已含） vs 收入 → 真實項目淨利
  • 項目結案：對比編預算時各分項 vs 實際支出，超支 / 節省的原因留記錄供下次預估

給 Ray + Shawn 拍板的 4 個問題

需要你跟 Shawn 一起拍板

  1. 確認推 B 雙軌制？　→ 是 / 否 / 想做 C 嚴格版
  2. 員工申請時「關聯項目」是可選還必填？　建議：可選。理由：不是所有費用都跟項目有關（內部培訓 / onboarding / 跨項目拜訪），必填會逼員工亂掛。
  3. 哪些費用類別會掛項目？　建議：
    • ② 供應商請款 → 幾乎都掛項目（工廠 PO / 樣品費 / 場地費 / 外包設計費 都是因項目而生）
    • ③ 項目員工差旅 → 掛項目（這就是要解的問題本身）
    • ① 個人報銷 → 不掛項目（內部會議 / onboarding / 一般辦公差旅 → 走 OPEX）
  4. 超預算門檻 — 多少要升級 Ray？　建議：
    • 該項目對應池還有額度 → 楊子核准
    • 該項目對應池不夠 → 強制升級 Ray（不分金額）
    • 整筆 > ¥10,000 同時項目池超 80% → 額外通知 Shawn 一聲
