試用啟動說明

Hi 各位，我們的內部 ERP 系統開始試用了。
這份文件 5 分鐘看完 — 你會知道怎麼登入、第一週要做什麼、撞到問題找誰。

一、為什麼開始試

過去我們的工作流程散在飛書 / 群組訊息 / 各種 Excel — 每次想知道「這件事到哪一棒了」都要問人。

新 ERP 把全公司接力的事整成一條鏈：業務員開單 → 安然把關 → Ray 拍板 → 採購 / 物流 → 收款 → 入帳。每一棒的人在自己的 lobby 看「我該做什麼」，做完點一下就送下一棒。

試用一週，我們會收集大家用得卡 / 不順 / 看不懂的地方，下週繼續改。

二、怎麼進去

🔗 raychou6913-tech.github.io/wujie-briefing/login.html

你的個人帳號 Ray 會單獨給你（用飛書私訊 / Email）。每個帳號看到的是你個人的工作台，跟別人的不一樣。

第一次登入的小提示
打開 lobby 後，中間會有一個橘色框框 — 那是 MJ（AI 助理）跟你打招呼，告訴你「你的主場是什麼、哪些工具在 sidebar」。看完點「✓ 知道了」就 dismiss。

系統剛開始跑，你的 inbox 可能是空的，這很正常。等別人開始用之後，事情會慢慢進來。

三、第一週要做什麼

打開 lobby，按一遍 sidebar 看看
左邊 sidebar 是你能做的事。**不會弄壞**，每個按鈕點下去都是 modal（彈窗），不會跳走。摸熟介面是第一步。
真的有工作就在 ERP 做一次
不要兩邊跑（一邊 ERP、一邊飛書群組）。試用期就是要看大家用得順不順 — 用得卡的地方就回報。
看不懂就問 MJ（右下角橘色 🤖）
不確定點哪、不懂某個欄位什麼意思、想知道某個流程 — 直接打字問。MJ 是 AI 不會嫌你問笨問題。
撞到 bug / 用得卡 → 按右下角「🐛 使用回報」
一句話也行。Ray + MJ 會看到，一週後我們會總整 review。越早講越能改。

五、紀律 · 試用期最重要的 3 件事

紀律 #1
不怕弄壞 — 試用期數據可以亂填，發現是測試會清掉。重要的是真的試一次。
紀律 #2
用得卡馬上回報，不要忍 — 別在心裡抱怨「這系統真難用」就跑回飛書群組。按右下角 🐛 一句話回報就行。
紀律 #3
一週後我們會 review 大家的回報 — 你的反饋會真的影響下週改什麼。不是丟到黑洞。

六、遇到問題找誰

  • 系統 bug / 用法不懂 → 右下角「🐛 使用回報」一句話送出
  • 緊急問題 / 系統打不開 → 直接 ping Ray（或 MJ）
  • 業務上的疑問（譬如「這個訂單要不要送 Ray」）→ 該問的主管不變（業務 → 安然、採購 → 賴哥）

七、隱私 · 機密

  • 系統只給蕪解 15 人內部用，外部看不到
  • 每個人看到的是自己的工作台 + 跟自己有關的事，不是全公司資料攤平
  • 對賭具體數字 / 法人結構 / 跨境款項 — 這些只有管理層帳號可見
