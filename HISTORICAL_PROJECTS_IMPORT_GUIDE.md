# 歷史專案匯入指南（Shawn 上線當天用）

> gg 的「斷代史原則」：上線只建「進行中」或「合約有效 MG 沒扣完」的歷史項目。
> 已結案／MG 已用完／合約過期的死案**不要建**。

---

## 為什麼這份指南存在

ERP 從 2026 年中開始記帳，但公司過去 15 年的歷史不能全塞進系統 —
塞太多會把整個 baseline 拖累、信任度崩潰。

**原則：只匯入「還在跑」的東西。**

---

## 哪些要建（白名單）

打開 `erp-project-new.html` 用 3 段確認流程建立。每個項目花 5 分鐘填完。

### ✅ 進行中的快閃店
- 場地正在準備、開店中、或還沒撤場的
- 例：Siam Paragon Mofusand（2026/3 開店中）、Terminal 21 Snoopy（籌備中）

### ✅ 合約還有效 + MG 還沒扣完的 IP 項目
- 5 個簽約 IP：Mofusand / Snoopy / Opanchu / Muziktiger / 飯糰梅子
- 每個 IP 在每個渠道有多個產品線，產品線級的開到「項目排期」
- 例：Mofusand Q4 大陸批發、Mofusand 曼谷快閃、Mofusand 台灣寄售⋯⋯

### ✅ 還沒結款的批發訂單
- 客戶下單還沒出貨的、出貨還沒收款的
- 例：上海潮玩 ¥150 萬 KA 訂單（5/06 下單，未出貨）

### ✅ Year 1 VAM baseline（你 3 月會議排的 10 筆預算）
- **已經在飛書「預算管理」表了**，不用重建
- 只是要讓每筆對應到一個新建的「項目排期」record

---

## 哪些**不要**建（黑名單）

### ❌ 已結案的快閃店
- 排球少年北京 ¥346 萬（已結案）→ 不用建
- JJK 曼谷 USD$187K（已結案）→ 不用建
- 這些歷史業績進 demo 數據錨點即可，不進 ERP 項目

### ❌ MG 已用完的 IP
- 過去合作但版稅金額已扣到 0 的 → 不用建

### ❌ 合約已過期的 IP
- 過去 15 年合作過的 30+ IP（Pikachu / Godzilla / LINE / POP MART⋯⋯）→ **全部不要建**
- 這些是公司歷史光環，不是 ERP 待辦

### ❌ 已收完款的批發訂單
- 過去做完已結款的 → 算公司歷史業績，但 ERP 不追蹤

## 建項目時填什麼

每個項目填這 7 個欄位（其他可選填）：

| 欄位 | 怎麼填 |
|------|------|
| **項目名稱** | 「IP + 渠道 + 季/數量」例：「Mofusand Q4 大陸批發」 |
| **類別** | 商品批發 / 快閃店 / IP商品授權 / 其他 |
| **IP 品牌** | 從下拉選（5 個真實 IP），未簽約用「探索中 IP」 |
| **負責人** | Ray / 安然 / Shawn 等 |
| **開始日期** | 項目啟動日（如：合約簽約日 / 第一筆採購日）|
| **預計收入** | 整個項目週期內預期能收的錢 |
| **預計支出** | 含生產、版稅、物流、攤提一次性費用 |

幣別 + 鎖定匯率系統會自動帶（用當下匯率），可手動覆寫。

## 建完之後

每個項目狀態都會是「**待 Ray 確認**」 — 這是 gg 設計的「項目錨點驗證」：
1. Shawn 建項目 → 狀態：待 Ray 確認
2. Ray 在戰情室 review baseline → 點過後升級為「baseline 鎖定」
3. 之後 VAM 數字才算 baseline，員工看到才有公信力

**前 3 個項目 Ray / Shawn 一定要面對面 review 過再 lock**。
之後類似的可以快速確認。

| 範圍 | 預估 |

| 5-10 個未結款批發訂單 | 30 分鐘 |
