## 系統架構

```
4 個爬蟲池（並行抓取）
    ↓
過濾層（熱度閾值 + 新穎度 + 適配性評分）
    ↓
Gemini Vision 圖像分類（哪些能掛 IP）
    ↓
飛書「供應鏈新品」表
    ↓
週推 Top 10 → 設計團隊（Bevis / 詩彤 / 珮綾）+ 賴哥 / 木子
    ↓
季推「趨勢報告」→ Ray review
```

## 過濾機制（強制執行，呼應 gg 要求）

| 過濾層 | 規則 | 阻擋率（預估）|
|--------|------|---------------|
| **熱度閾值** | 搜尋量月成長 >100% 才推 | 90% 阻擋 |
| **新穎度** | 3 個月內新上架，IP 圈尚未大量出現 | 50% 阻擋 |
| **適配性評分** | Gemini Vision 判斷「能不能掛 IP」（太大件 / 太貴 / 太特殊用途剃掉）| 30% 阻擋 |
| **價格區間** | 零售價 50-500 RMB 之間（外圍視 IP 而定）| 20% 阻擋 |

從爬到 1000 個 → 過濾後剩 20-50 個 → 週推 Top 10。

## 推送節奏

- **週一 9:00**：自動推飛書 + LINE 通知設計團隊
- **每週 Top 10**：簡短 brief，每個 1 段 + 圖
- **每季度報告**：Ray review，看哪個池真的有產出

## 不做的事

- ❌ **每日推送 100 個連結** — 賴哥木子會崩潰
- ❌ **無過濾搬運** — 違反 gg 要求
- ❌ **抓電商評論文字** — 太多噪訊
- ❌ **抓爆紅商品** — 那是「已熱」不是「上升」，沒先機價值
