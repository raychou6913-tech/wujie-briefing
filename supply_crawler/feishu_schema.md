# 飛書「供應鏈新品」表 schema

> 待 MJ 在飛書建表（5/15 跟賴哥/木子對齊後）

---

## 表格資訊

- **表名**：供應鏈新品
- **Base AT**：`Sg7ObNELHaAdZzshGIFjWPVhpce`（ERP 主 Base）

## 欄位設計

| 欄位 | type | 選項 / 說明 |
|------|------|------------|
| 池 | SingleSelect (3) | A 快閃載體 / B IP周邊 / C 上升趨勢 / D 新工法 |
| 子分類 | SingleSelect (3) | 依池細分（A1-A3 / B1-B7 / C1-C7 / D1-D5）|
| 商品名稱 | Text (1) | 爬蟲帶入 |
| 來源平台 | SingleSelect (3) | Etsy / Amazon / 淘寶 / Rakuten / 1688 / Pinterest / TikTok / 小紅書 |
| 來源 URL | URL (15) | 連結 |
| 商品圖 | Attachment (17) | 1-3 張代表圖 |
| 價格區間 | Text (1) | 譬如「50-200 RMB」 |
| 熱度信號 | Text (1) | 月成長率 / 評論數 / 收藏數 |
| 熱度成長 % | Number (2) | 月成長率（譬如 150 代表 +150%）|
| 新穎度評分 | Number (2) | 1-10 由 Gemini Vision 判斷 |
| 適配性評分 | Number (2) | 1-10 由 Gemini Vision 判斷能不能掛 IP |
| 推送週次 | Text (1) | 譬如「W20-2026」|
| 推送 Top 排名 | Number (2) | 1-10 |
| 設計團隊評語 | Text (1) | 設計團隊看完的感想（人工填）|
| 採購評語 | Text (1) | 賴哥/木子看完（人工填）|
| 狀態 | SingleSelect (3) | 新進 / 觀察中 / 已嘗試打樣 / 已採用量產 / 已退場 |
| 對應項目 | Text (1) | 若採用，link 到飛書「項目排期」record_id |
| 爬取時間 | DateTime (5) | |

## 為什麼這樣設計

- **池 + 子分類**：快速過濾按用途
- **熱度信號 + 成長 %**：強制呼應 gg 過濾要求
- **新穎度 + 適配性評分**：Gemini Vision 自動算
- **推送週次 + Top 排名**：追蹤每週推送內容
- **設計團隊評語 / 採購評語**：人工 review 落點
- **狀態 → 對應項目**：閉環，從爬到產品

## 預期記錄量

| 階段 | 月寫入 |
|------|--------|
| Phase 1 建構（5 月底前）| 0-100 筆（測試）|
| Phase 1 跑起來（6 月起）| 月新增 50-100 / 月推送 40-80 |
| Phase 2（9 月起）| 月新增 100-200 / 月推送 60-120 |

## 推送格式（飛書推 + LINE 通知）

每週一 9:00 自動推：
- 飛書：表內 filter「推送週次 = 本週」+「池 = 各池」分組
- LINE：簡短訊息「本週供應鏈巡邏 — 池 A 5 個、池 B 15 個、池 C 10 個、池 D 5 個 → [飛書連結]」

## 季度報告（自動生成 → Ray review）

每季底自動跑：
- 各池本季推送量
- 設計團隊採用率
- 採購打樣 → 量產轉換率
- Ray review 寫 1-2 句決策（加碼 / 維持 / 調整）

## 寫入 API

跟現有 ERP 同 pattern，先用 curl 測試手動匯入幾筆，跑通後接爬蟲自動寫入。
